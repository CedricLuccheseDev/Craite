# Commit Command

Analyze the current git changes and create a commit with a descriptive message.

## Instructions

1. Run `git status` to see staged and unstaged changes
2. Run `git diff --cached` to see staged changes (if any), or `git diff` for unstaged changes
3. Get the current branch name with `git branch --show-current`
4. Analyze the changes and write a concise one-line commit message in English

## Commit Message Format

```
branch_name: brief description of changes
```

## Rules

- The description should be one sentence, concise and clear
- Use imperative mood (e.g., "fix", "add", "update", "remove")
- Do NOT add "Generated by" or "Co-authored-by" lines
- Do NOT add any footer or trailer to the commit message
- Stage all relevant changes before committing
- Use the exact branch name as prefix

## Example

For branch `fix/688` with changes fixing a bug in counting module:
```
fix/688: fix map display issue when editing counting request
```

## Execution

After analyzing, stage the changes and create the commit with the generated message.
