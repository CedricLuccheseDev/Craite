# Commit Command

Analyze the current git changes and create a commit with a conventional commit message.

## Instructions

1. Run `git status` to see staged and unstaged changes
2. Run `git diff --cached` to see staged changes (if any), or `git diff` for unstaged changes
3. Analyze the changes and determine the correct commit type
4. Stage all relevant changes and create the commit

## Commit Message Format

```
<type>: <brief description>
```

## Commit Types

| Type       | When to use                                      |
|------------|--------------------------------------------------|
| `feat`     | New feature or functionality                     |
| `fix`      | Bug fix                                          |
| `refactor` | Code restructuring without behavior change       |
| `docs`     | Documentation only                               |
| `style`    | Formatting, CSS, no logic change                 |
| `perf`     | Performance improvement                          |
| `test`     | Adding or updating tests                         |
| `chore`    | Dependencies, config, tooling (hidden in changelog) |

For breaking changes, add `!` after the type: `feat!: redesign sample model`

## Rules

- The description should be one sentence, concise and clear
- Use imperative mood (e.g., "fix", "add", "update", "remove")
- All in English, lowercase after the colon
- Do NOT add "Generated by" or "Co-authored-by" lines
- Do NOT add any footer or trailer to the commit message
- Stage all relevant changes before committing
- Pick the type that best matches the **primary intent** of the changes
- If changes span multiple types, use the most significant one

## Examples

```
feat: add background scanning for sample libraries
fix: resolve login redirect loop on first launch
refactor: extract watcher logic into dedicated composable
docs: add CI pipeline documentation
chore: update pnpm dependencies
feat!: redesign sample model with new category system
```

## Execution

After analyzing, stage the changes and create the commit with the generated message.
